# 安全与隐私策略（SECURITY）

> 对应 skill 版本：**v2.1（Spider_XHS 版）**  
> 最后更新：2026-05-14

本文档说明 blogger-distiller 在**数据安全**与**隐私处理**上的具体做法，以及遇到安全问题的上报渠道。

---

## 一、四条核心承诺

### 承诺 1：零账号风险

- ✅ 不模拟登录、不注入 Cookie、不使用任何平台账号凭证
- ✅ 全程通过 Spider_XHS 本地爬虫获取公开数据
- ✅ 你的账号**零封号风险**（工具本身层面）

### 承诺 2：纯本地运行

- ✅ 采集的数据仅保存在**你自己的电脑**上（`data/` 目录）
- ✅ 不上传、不回传、不埋点、不统计任何用户行为
- ✅ 作者的服务器上**没有任何**使用者数据

### 承诺 3：默认脱敏

- ✅ 评论者身份**默认脱敏**（读者1 / 读者2 / 作者）
- ✅ 评论者的 userId / 昵称 / 头像 / IP 属地**不保存**
- ✅ 评论正文保留（作为内容研究素材）
- ✅ 脱敏逻辑集中在 `scripts/utils/privacy.py`，开源可审计

### 承诺 4：最小化 API 调用

- ✅ 本地爬虫，不加压小红书服务器
- ✅ 主页笔记不足时自动触发多关键词搜索补充，避免无效重复调用

## 二、数据处理清单

| 数据类型 | 来源 | 存储位置 | 默认处理 |
|---------|------|---------|---------|
| 博主主页信息 | Spider_XHS | 本地 `data/*/raw/` | 原样保存 |
| 博主笔记正文 | Spider_XHS | 本地 `data/*/raw/` | 原样保存（博主本人已公开内容） |
| 博主笔记封面图 URL | Spider_XHS | 本地 `data/*/raw/` | 仅保存 URL，不下载图片 |
| 评论正文 | Spider_XHS | 本地 `data/*/raw/` | **保留**（研究素材） |
| 评论者昵称 | Spider_XHS | ❌ 不保存 | 脱敏为"读者N" |
| 评论者 userId | Spider_XHS | ❌ 不保存 | 脱敏丢弃 |
| 评论者头像 | Spider_XHS | ❌ 不保存 | 脱敏丢弃 |
| 评论者 IP 属地 | Spider_XHS | ❌ 不保存 | 脱敏丢弃 |
| Cookie | 用户本地 `.env` 文件 | 仅在内存中 | 不写入 `data/`，不上传日志 |

## 三、风险与缓解

| 风险 | 可能性 | 缓解措施 |
|------|--------|---------|
| Spider_XHS 本地爬虫失效 | 中 | 跟随 Spider_XHS 项目更新 |
| 小红书平台政策变更 | 中 | Cookie 过期提示 + 更新指引 |
| 数据质量不一致 | 低 | `quality.py` 数据质量检查 |
| 用户误将 Cookie 提交到 Git | 低 | 默认通过 `.env` 文件读取，`.gitignore` 已排除 |
| 用户误分享含个人信息的报告 | 低 | 评论者默认脱敏；报告中不出现评论者可识别信息 |
| 本地 `data/` 目录被他人访问 | 低 | 由使用者自行管理本地文件权限 |

## 四、上报渠道

### 4.1 常规 Bug 或功能建议

在 GitHub 开 Issue 即可：  
🔗 [github.com/otter1101/blogger-distiller/issues](https://github.com/otter1101/blogger-distiller/issues)

### 4.2 涉及隐私 / 安全的敏感议题

**请勿公开发在 Issue 区**。使用 GitHub Security Advisory 私下提交：  
🔗 [github.com/otter1101/blogger-distiller/security/advisories/new](https://github.com/otter1101/blogger-distiller/security/advisories/new)

适用情况包括但不限于：

- 发现代码中存在隐私泄露风险
- 发现本工具被他人用于恶意场景
- 希望申请从项目公开材料（README / 示例数据 / 截图）中移除涉及你的可识别信息
- 其他不适合公开讨论的安全议题

### 4.3 响应时效

作者是个人维护者，不是公司，响应时效**尽力而为**：

- 一般 Bug：视优先级安排
- 隐私/安全敏感议题：**优先处理**（通常 7 天内给首次响应）

## 五、能做到 vs 做不到

**能做到**（在项目公开材料范围内）：

- ✅ 从 README / 示例数据 / 截图中移除涉及你的可识别信息
- ✅ 在代码层面修复已披露的隐私泄露问题
- ✅ 在 SKILL.md / DISCLAIMER.md 中明确禁止某类使用场景

**做不到**（诚实说明）：

- ❌ 无法删除已被使用者下载到本地电脑的数据（工具是纯本地运行的）
- ❌ 无法撤销使用者已经分享出去的蒸馏报告
- ❌ 无法替使用者的行为负法律责任

这是一个**个人维护的开源工具**，不是商业数据服务。请用符合它定位的期望使用它。
