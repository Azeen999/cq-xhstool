# 粗趣活动运营 SOP

> 活动全流程标准化操作手册

---

## 活动前准备清单

### 时间节点

| 时间 | 事项 | 负责人 |
|------|------|--------|
| 活动前 7 天 | 确定活动方案、定场地、定档期 | 运营 |
| 活动前 5 天 | 完成活动海报/物料设计 | 设计 |
| 活动前 5 天 | 发布小红书招募帖 | 运营 |
| 活动前 3 天 | 开始建群、拉已报名用户 | 运营 |
| 活动前 1 天 | 确认场地、准备物料、群内提醒 | 运营 |
| 活动当天 | 提前 30 分钟到场布置 | 带场人 |
| 活动后 2 小时 | 发活动回顾、群内感谢 | 运营 |
| 活动后 1 天 | 发小红书回顾帖 | 运营 |

### 物料清单

- [ ] 签到表/二维码
- [ ] 桌游/活动道具（提前检查完整性）
- [ ] 零食饮料（按人数 1.2 倍准备）
- [ ] 话题卡/互动道具
- [ ] 相机/三脚架（用于拍摄）
- [ ] 名牌贴纸
- [ ] 备用（纸巾、充电宝、应急药品）

### 场地确认 checklist

- [ ] 桌椅布局是否适合活动类型
- [ ] 灯光音响是否正常
- [ ] WiFi 是否正常
- [ ] 空调/通风是否正常
- [ ] 洗手间位置确认
- [ ] 消防通道确认

## 活动招募节奏

### 最佳发布时间

| 内容类型 | 发布时间 | 理由 |
|----------|----------|------|
| 活动招募帖 | 周三/周四 20:00 | 提前 3-5 天，用户规划周末 |
| 活动回顾帖 | 周日/周一 12:00 | 活动结束后复盘 |
| 干货/情绪内容 | 周二/周五 18:00 | 下班通勤刷手机高峰 |

### 招募渠道优先级

1. **小红书** — 主要渠道，发招募帖+活动海报
2. **微信群** — 老用户群内预告+报名链接
3. **朋友圈** — 运营个人号朋友圈发布
4. **公众号** — 每周活动合集汇总

### 报名确认流程

1. 用户在小程序/链接报名付费
2. 系统自动发送确认通知
3. 运营手动拉群（活动前 3 天）
4. 群内发送活动须知+地址指引
5. 活动当天早上群内提醒

### 活动取消管理

- 报名不足最低人数 → 活动前 24 小时通知取消
- 全额自动退款，无需用户操作
- 发送替代活动推荐

## 活动后复盘

### 反馈收集

活动结束后发送问卷，收集维度：

- [ ] 整体满意度（1-5 分）
- [ ] 带场人评分（1-5 分）
- [ ] 场地评分（1-5 分）
- [ ] 最满意的环节
- [ ] 改进建议
- [ ] 下次想参加的活动类型

### 回顾帖模板

```
📸 活动精彩瞬间
（3-5 张活动现场照片）

👥 参与者说
「XXX」—— 参与者小A
「XXX」—— 参与者小B

🎉 本期亮点
- {亮点1}
- {亮点2}

🔜 下期预告
{活动名称}即将开放报名，持续关注！

#粗趣 #深圳周末 #深圳社交
```

### 数据记录

每次活动记录以下数据：

- 活动类型、日期、场地
- 报名人数、实际参与人数、取消人数
- 收入（报名费）vs 支出（场地+物料+人力）
- 小红书帖子的曝光/互动数据
- 参与者反馈评分
